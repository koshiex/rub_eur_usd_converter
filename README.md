# Конвертер RUB → EUR / USD

## Назначение проекта

Веб-приложение для расчёта итоговой суммы при переводе российских рублей (RUB) в евро (EUR) или доллары США (USD) через цепочку платёжных систем и банков. Приложение помогает оценить, сколько в итоге получит отправитель/получатель после всех конвертаций и комиссий по двум типичным маршрутам, использующим карту МИР РФ, белорусский БНБ-Банк и кошелёк Pyypl.

Приложение отображает актуальные (по возможности) курсы с пометкой об источнике и статусе получения (успешно/использован резервный курс).

## Сценарий 1: RUB (карта РФ, МИР) → BYN (BNB Belkart) → EUR (счёт/карта BNB)

Маршрут перевода:

1. Пользователь вводит сумму в RUB, списываемую с российской карты МИР.
2. Средства переводятся на карту Белкарт БНБ-Банка (Беларусь). Конвертация RUB → BYN происходит по курсу платёжной системы МИР.
3. На стороне БНБ-Банка сумма в BYN конвертируется в EUR по внутреннему курсу счёта/аккаунта банка (не карточный курс) и зачисляется на EUR-счёт или EUR-карту БНБ.

Результат сценария — сумма в **EUR**, которую фактически получит пользователь на счёте/карте BNB после двойной конвертации RUB → BYN → EUR.

## Сценарий 2: RUB → BYN → EUR (BNB) → USD (Pyypl)

Маршрут перевода:

1. Шаги 1–3 совпадают со Сценарием 1: RUB с карты МИР → BYN на Белкарт БНБ → EUR на счёт/карту BNB.
2. Далее EUR с карты/счёта BNB пополняется кошелёк Pyypl в USD. Конвертация EUR → USD выполняется по курсу Visa (карта выпуска BNB — Visa).
3. При пополнении Pyypl удерживается комиссия сервиса.

Результат сценария — сумма в **USD**, зачисляемая на кошелёк Pyypl после конвертаций RUB → BYN → EUR → USD и вычета комиссии Pyypl.

## Сценарий 3: RUB → EUR транзакция через Pyypl (USD → EUR)

Маршрут:

1. Шаги 1–3 совпадают со Сценарием 2: RUB → BYN → EUR → USD с зачислением на кошелёк Pyypl после комиссии top up.
2. С кошелька Pyypl (USD) производится покупка или транзакция в EUR: Pyypl удерживает **card transaction fee** (по умолчанию **$0.00 + 5%**), а затем конвертирует оставшиеся USD в EUR по внутреннему курсу.
3. В калькуляторе курс USD → EUR берётся как инверсия динамического курса Visa EUR → USD (`1 / visaEurToUsd`) — этого достаточно для оценки; при желании можно поменять комиссию в UI.

Результат сценария — сумма в **EUR**, которую получит продавец/провайдер после цепочки RUB → BYN → EUR → USD (Pyypl) → EUR (покупка).

В интерфейсе отображаются промежуточные значения:

- USD на Pyypl после top up (результат Сценария 2);
- удержанная Pyypl card transaction fee в USD;
- USD, доступные для FX после комиссии;
- итоговая сумма EUR-транзакции.

## Обратный расчёт: сколько RUB нужно на старте

В интерфейсе ниже прямых сценариев есть блок **«Обратный расчёт: сколько RUB нужно»**. Он отвечает на обратный вопрос: «Сколько нужно положить на старте RUB, чтобы оплатить покупку на заданную сумму?». Реализованы три обратных сценария, симметричных прямым.

Обозначения:

- `mirRubToByn` — сколько BYN получит пользователь за 1 RUB по курсу МИР.
- `bnbBynPerEur` — сколько BYN стоит 1 EUR по курсу БНБ (счёт).
- `visaEurToUsd` — сколько USD дают за 1 EUR по курсу Visa.
- `topUpFixed`, `topUpPercent` — комиссия Pyypl за top up (USD, %).
- `cardFixed`, `cardPercent` — комиссия Pyypl за card transaction (USD, %).

### Обратный Сценарий 1: нужно `N EUR` на счёте/карте BNB

```
bynNeeded = N_EUR × bnbBynPerEur
rubNeeded = bynNeeded / mirRubToByn
```

### Обратный Сценарий 2: нужно `N USD` на Pyypl после top up

```
usdBeforeFee = (N_USD + topUpFixed) / (1 − topUpPercent/100)
eurNeeded    = usdBeforeFee / visaEurToUsd
bynNeeded    = eurNeeded × bnbBynPerEur
rubNeeded    = bynNeeded / mirRubToByn
```

Минимум Pyypl по top up — **20 USD до комиссии**: если `usdBeforeFee < 20`, интерфейс показывает предупреждение и минимально необходимую сумму в RUB, чтобы пополнение было в принципе возможно.

### Обратный Сценарий 3: покупка `N EUR` с Pyypl USD

Инверсия цепочки «EUR BNB → USD Pyypl (top up fee) → USD после card transaction fee → EUR (инверсия Visa)».

```
usdForFx      = N_EUR × visaEurToUsd               // USD, который уйдёт на FX-покупку
usdAfterTopup = (usdForFx + cardFixed) / (1 − cardPercent/100)
usdBeforeTopup = (usdAfterTopup + topUpFixed) / (1 − topUpPercent/100)
eurNeeded     = usdBeforeTopup / visaEurToUsd
bynNeeded     = eurNeeded × bnbBynPerEur
rubNeeded     = bynNeeded / mirRubToByn
```

Также проверяется, что `usdBeforeTopup ≥ 20` (минимум top up Pyypl). Если курс какого-либо звена равен нулю или отсутствует, блок показывает предупреждение об отсутствии курсов; в расчётах используется безопасное деление, чтобы не появлялись `Infinity`/`NaN`.

## Источники курсов

- **RUB → BYN (МИР):** курс платёжной системы МИР, получаемый с сайта [onlymir.ru](https://onlymir.ru) / открытых данных МИР.
- **BYN → EUR (БНБ-Банк):** курсы БНБ-Банка ([bnb.by](https://bnb.by), `imbank`/счёт), используются курсы **счёта/аккаунта**, а не карточные курсы.
- **EUR → USD (Visa):** попытка получения курса через Visa FX Calculator / Visa FX API. При недоступности API используется резервный (fallback) курс; в интерфейсе отображается статус источника курса.

## Настройки комиссии Pyypl

### Top up (Сценарий 2)

- Комиссия за пополнение кошелька Pyypl по умолчанию: **$0.50 + 3.5%** от суммы пополнения (Load from Debit Card).
- Минимальная сумма пополнения до вычета комиссии: **20 USD**.

Значения настраиваются в интерфейсе, при расчёте итоговой суммы в USD приложение применяет формулу: `USD_получено = (USD_до_комиссии − 0.50) × (1 − 0.035)`, при условии соблюдения минимальной суммы.

### Card transaction (Сценарий 3)

- Комиссия за покупку/транзакцию по карте Pyypl по умолчанию: **$0.00 + 5%** (International Purchase / Local Purchase).
- Значения настраиваются в интерфейсе полями `Pyypl card fixed, USD` и `Pyypl card percent, %`.

Формула расчёта: `USD_для_FX = USD_Pyypl − (USD_Pyypl × 5% + 0.00)`, затем `EUR_итог = USD_для_FX × (1 / курс_Visa_EUR→USD)`.

## Локальная разработка

### Требования

- [Node.js](https://nodejs.org/) (LTS рекомендуется)
- npm

### Установка и запуск

```bash
npm install           # установка зависимостей
npm run dev           # запуск dev-сервера
```

После запуска приложение доступно по адресу: <http://localhost:5000>.

### Дополнительные команды

```bash
npm run check         # проверка типов TypeScript (tsc)
npm run build         # сборка production-бандла
npm run start         # запуск собранного приложения (NODE_ENV=production)
```

## Запуск в Docker

Для production-развёртывания в репозитории есть многостадийный `Dockerfile` (сборка клиент + сервер, прод-зависимости, запуск от непривилегированного пользователя). Приложение слушает порт **5000**.

```bash
docker build -t rub-converter .
docker run --rm -p 5000:5000 rub-converter
```

После запуска приложение будет доступно по адресу <http://localhost:5000>.

Переменные окружения можно передать через `--env-file`:

```bash
docker run --rm -p 5000:5000 --env-file .env rub-converter
```

## Динамические курсы и статус источников

Курсы валют подтягиваются динамически из внешних источников (МИР, БНБ-Банк, Visa FX). Если внешний сервис временно недоступен или возвращает ошибку, приложение использует **резервный (fallback) курс** и явно отображает в интерфейсе статус каждого источника: успешное получение актуального курса либо использование резервного значения. Это позволяет пользователю понимать, насколько свежие данные легли в итоговый расчёт.
